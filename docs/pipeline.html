Qualidade de Código

SonarQube — Análise Estática

Cada serviço é analisado individualmente. O Quality Gate bloqueia o pipeline se os critérios mínimos não forem atendidos.

📊 Métricas analisadas

Cobertura de testes ≥ 70%
Code Smells monitorado
Bugs zero tolerância
Vulnerabilidades zero tolerância
Duplicações monitorado
Security Hotspots revisão

🚦 Quality Gate — como funciona

Ao final da análise, o SonarQube avalia se o projeto atende às condições definidas.

Se passar: pipeline continua para o Docker build.

Se reprovar: o step falha com exit code 1, o pipeline é marcado como ❌ Failed e o PR não pode ser mergeado.

O parâmetro -Dsonar.qualitygate.wait=true garante que o scanner aguarda o resultado antes de retornar.
# sonar-project.properties — exemplo (api-gateway)
sonar.projectKey=pooker-api-gateway
sonar.projectName=Pooker - API Gateway
sonar.sources=src/main/java
sonar.tests=src/test/java
sonar.coverage.jacoco.xmlReportPaths=build/reports/jacoco/test/jacocoTestReport.xml
Infraestrutura

Componentes da esteira

Toda a infraestrutura de CI/CD roda localmente, sem custo de cloud.

⚡
GitHub Actions Runner
Self-hosted runner instalado nativamente no Windows. Fica idle aguardando jobs e executa as pipelines localmente.
🔍
SonarQube Community
Rodando em Docker Desktop na porta 9000. Recebe os relatórios do sonar-scanner e avalia o Quality Gate.
🐳
Docker Desktop
Hospeda o SonarQube e o PostgreSQL. As imagens dos serviços são buildadas ao final de cada pipeline bem-sucedida.
🌐
ngrok Tunnel
Expõe o SonarQube local (localhost:9000) via URL pública HTTPS, permitindo acesso remoto ao dashboard de qualidade.
Gatilhos

Quando as pipelines disparam

As pipelines usam filtros de path para executar apenas quando código relevante é alterado.

☕ Java Pipeline dispara quando:
backend/api-gateway/**
backend/auth-service/**
🐍 Python Pipeline dispara quando:
backend/filmes-service/**
backend/alugueis-service/**
backend/user-service/**
backend/payment-service/**
🅰️ Angular Pipeline dispara quando:
frontend/**
Também disponível via workflow_dispatch para execução manual.
